# FluidSelf

> 厌倦了每次新对话都要向Agent重申你的喜好？
> 让Agent牢记你的喜好，随时调频，随时生效。
> 不做性格诊断，不贴人格标签，只适配你的工作方式。

## 这是什么？

**FluidSelf** 是一套通用 Agent 行为规则，让 Agent 根据你的协作风格偏好动态调整沟通方式。

使用用 **6 条行为轴** 精确描述你在不同工作场景下的偏好，并在每一次对话中静默适配。

### 核心理念

| 其他方式 | FluidSelf 的方式 |
|---------|-----------------|
| 贴固定标签（「你是 XXX」） | 只记录行为偏好，绝不输出标签 |
| 一次设定终身定性 | 每次任务都可微调，风格是流动的 |
| 16 种固定类型 | 6 条行为轴 × 2 极 = 64 种组合 |

## 快速开始

### 安装

FluidSelf 是一套通用 Agent 行为规则，不绑定任何特定平台。

**方式一：作为规则文件（推荐）**

将 `FluidSelf_CN/` 目录复制到你的 Agent 的规则/技能目录：

| 使用场景 | 操作 |
|---------|------|
| WorkBuddy | 放入 `~/.workbuddy/skills/` |
| Cursor | 放入 `.cursor/rules/`，或直接在 Rules 面板添加 |
| Claude Code | 放入 `~/.claude/` 并引用 `SKILL.md` |
| CodeBuddy Code | 放入 `~/.codebuddy/skills/` |
| 通用 Agent | 将 `SKILL.md` 和 `references/` 作为系统提示词注入 |

**方式二：通过 Git**

```bash
git clone https://github.com/SimonCoreLab/fluidself.git
cp -r fluidself/FluidSelf_CN/ ~/.workbuddy/skills/fluidself/  # 替换为你使用的平台路径
```

### 首次使用

FluidSelf 会自动检测你是否有配置。首次使用时，会进行 4 轮轻量 A/B 选择（不填问卷感）：

1. 「更在意整体框架」还是「先关注具体事实」？
2. 「优先进行逻辑分析」还是「优先输出价值感受」？
3. 「有明确计划再执行」还是「习惯边做边探索」？
4. 「少问多做直接交付」还是「关键节点确认对齐」？

4 轮结束后，Agent 即按你的偏好运行。随时说「换个方式」就能调整。

### 怎么调整风格

不需要记任何触发词。当你想调整时，直接说出你的感受就好：

> 「有点啰嗦」「先说结论」
> 「展开讲讲」「多说细节」
> 「我没看懂」「反向推导」
> 「换个方式试试」「两个版本对比」

Agent 会从你的表达里自动识别方向并调整。越自然越好。

## 架构：三阶段状态机

FluidSelf 不贴标签，像调频一样协助工作：听你说，记住你，越用越准。



```
Onboarding (Phase 0)     Working (Phase 1)          Reflection (Phase 2)
发现偏好                    适配风格                        持续微调
     │                          │                              │
     ▼                          ▼                              ▼
  Onboarding  ────────►      Working  ──────────►       Reflection
  四轮自然对话                按 6 轴调整输出             L1 任务后轻量反馈
  收敛 6 轴初始值             Domain 自动检测             L2 同域累计达间隔阈值（默认 3 次）后小结
                             不满信号实时修正               L3 确认后更新配置
                                │                            │
                                └────────────────────────────┘
                                      反馈循环（闭环）
```

## 6 条行为轴

每条轴是一个连续谱，Agent 根据你的偏好在 A/B 之间动态调整：

| # | 轴名 | A 侧 | B 侧 |
|---|-----|------|------|
| 1 | 信息偏好 | 抽象框架 + 概念优先 | 具体案例 + 细节优先 |
| 2 | 决策风格 | 逻辑分析 + 客观标准 | 价值感受 + 人际共识 |
| 3 | 行动方式 | 计划驱动 + 收束执行 | 探索驱动 + 开放灵活 |
| 4 | 互动习惯 | 少问多做 + 独立交付 | 多确认 + 逐段对齐 |
| 5 | 表达密度 | 精炼要点 | 丰富叙事 |
| 6 | 语调姿态 | 冷峻直接 | 温暖鼓励 |

6 轴独立取值，自由组合，形成 **64 种风格配置**。

> 当前会话的配置会记录在持久化记忆中，随时可以说「查看我的风格配置」来确认。

## Domain 自动适配

你不需要手动切换。Agent 会从你的输入中判断当前任务类型，自动微调风格倾向：

| 当前在做什么 | 什么场景 | Agent 会怎么调整 |
|------------|---------|-------------|
| 写代码 | 写代码、调试、架构设计 | 更精炼、逻辑优先、少废话 |
| 写文档 | 周报、文章、产品文案 | 更叙事、鼓励语气、多确认感受 |
| 分析思考 | 拆解问题、做决策 | 给框架、允许多路径、展示思考过程 |
| 沟通表达 | 回复消息、写汇报、对外表达 | 偏共识、确认对齐、提供多版本 |

## 核心特性

1. **覆盖完整**：内部有完整映射逻辑，外部只呈现行为维度
2. **反思可关闭**：L1/L2/L3 三级反思均可由你一键关闭
3. **Domain 自适应**：不同任务类型（代码/写作/分析/沟通）自动调整输出风格
